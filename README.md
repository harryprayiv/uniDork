# uniDork: postgresql-based movie library organization in pure Unison

I use Kodi to watch my movie collection. I curently use an overcomplicated hybrid of filebot and some bash scripts to populate my library and have been meaning to replace filebot in particular since it is closed source, proprietary, and simply scans open API's like tmdb for its info. My goal is to fully replace the filebot AMC script in its entirety eventually. Currently, this unison script just scans a folder for folders with nfo files then adds those to a database. It doesn't accomplish this yet actually, because I am doing some absurdly dumb things with memory. Unison is strange and I just keep crashing the ucm with my code.
